Pro-Tasker API | Backend Test Instructions | ()

RESTful backend API built with Node.js, Express, and MongoDB. Secure JWT authentication and ownership-based authorization for Projects and nested Tasks.

These instructions demonstrate how to test authentication, project, and task endpoints using a REST client.

Backend Status

  • Health check: GET http://localhost:3000/health | GET /health
  • API base: http://localhost:3000/api | /api

Expected: HTTP 200 response from /health

Not expected: 404 or server not responding

Base URL

  • Root URL: http://localhost:3000/
  • API URL: http://localhost:3000/api

If your server uses a different port, replace 3000 with your port.

Postman Testing Steps (12 Total)

1) Register User A

  • Method: POST
  • URL: http://localhost:3000/api/auth/register | /api/auth/register
  • Body: JSON
{
  "name": "Rocky",
  "email": "rocky@test.com",
  "password": "123456"
}

Expected: HTTP 201 and token returned

Not expected: HTTP 500 or missing token

2) Login User A (save TOKEN_A)

  • Method: POST
  • URL: http://localhost:3000/api/auth/login | /api/auth/login
  • Body: JSON
{
  "email": "rocky@test.com",
  "password": "123456"
}

Expected: HTTP 200 and token returned

Not expected: HTTP 401 with valid credentials

3) Protected Route Without Token (security proof)

  • Method: GET
  • URL: http://localhost:3000/api/projects | /api/projects
  • No Authorization header

Expected: HTTP 401 Unauthorized

Not expected: HTTP 200

4) Create Project With TOKEN_A (save PROJECT_ID)

  • Method: POST
  • URL: http://localhost:3000/api/projects | /api/projects
  • Headers: Authorization: Bearer TOKEN_A, Content-Type: application/json 
{
  "name": "Rocky Project",
  "description": "Ownership test project"
}

Expected: HTTP 201 and project _id returned

Not expected: HTTP 401

5) Get My Projects With TOKEN_A

  • Method: GET
  • URL: http://localhost:3000/api/projects | /api/projects
  • Header: Authorization: Bearer TOKEN_A

Expected: HTTP 200 and array of projects you own

Not expected: other users’ projects in the list

6) Get Single Project By ID With TOKEN_A

  • Method: GET
  • URL: http://localhost:3000/api/projects/PROJECT_ID | /api/projects/PROJECT_ID
  • Header: Authorization: Bearer TOKEN_A

Expected: HTTP 200 and project object

Not expected: 404 when PROJECT_ID is valid

7) Create Task Under Project With TOKEN_A (save TASK_ID)

  • Method: POST
  • URL: http://localhost:3000/api/projects/PROJECT_ID/tasks | /api/projects/PROJECT_ID/tasks
  • Header: Authorization: Bearer TOKEN_A
{
  "title": "Task A1",
  "description": "Created by User A",
  "status": "To Do"
}

Expected: HTTP 201 and task _id returned

Not expected: 404 route not found

8) Get Tasks For Project With TOKEN_A

  • Method: GET
  • URL: http://localhost:3000/api/projects/PROJECT_ID/tasks | /api/projects/PROJECT_ID/tasks
  • Header: Authorization: Bearer TOKEN_A

Expected: HTTP 200 and tasks array

Not expected: 403 for the owner

9) Register and Login User B (save TOKEN_B)

  • Register a second user with a different email
  • Login as the second user and save the returned token

Register User B

  • Method: POST
  • URL: http://localhost:3000/api/auth/register | /api/auth/register
  • Body: JSON
{
  "name": "User B",
  "email": "userb@test.com",
  "password": "123456"
}

Expected: HTTP 201 and token returned

Login User B

  • Method: POST
  • URL: http://localhost:3000/api/auth/login | /api/auth/login
  • Body: JSON
{
  "email": "userb@test.com",
  "password": "123456"
}

Expected: HTTP 200 and token returned (save as TOKEN_B)

Not expected: using the same email as User A

10) User B Tries to Access User A Project (ownership proof)

  • Method: GET
  • URL: http://localhost:3000/api/projects/PROJECT_ID | /api/projects/PROJECT_ID
  • Header: Authorization: Bearer TOKEN_B

Expected: HTTP 403 Forbidden

Not expected: HTTP 200

11) User B Tries to Access User A Tasks

  • Method: GET
  • URL: http://localhost:3000/api/projects/PROJECT_ID/tasks | /api/projects/PROJECT_ID/tasks
  • Header: Authorization: Bearer TOKEN_B

Expected: HTTP 403 Forbidden

Not expected: HTTP 200

12) Update a Task With TOKEN_A (CRUD proof)

  • Method: PUT
  • URL: http://localhost:3000/api/projects/PROJECT_ID/tasks/TASK_ID | /api/projects/PROJECT_ID/tasks/TASK_ID
  • Header: Authorization: Bearer TOKEN_A
  • Body: JSON
{
  "status": "Done"
}

Expected: HTTP 200 and updated task returned

Not expected: 403 for the owner
